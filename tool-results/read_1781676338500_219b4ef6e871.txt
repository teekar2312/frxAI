     1→# FinexFX AI Trading System — Worklog
     2→
     3→## Project Overview
     4→A comprehensive forex live-trading dashboard (Next.js 16 + TypeScript + shadcn/ui + Prisma/SQLite).
     5→Represents the monitoring & operational layer of a Python 3.14 + MT5 (FINEX Indonesia) scalping bot
     6→with AI/ML analysis. Pairs: EURUSD, USDJPY, GBPUSD, XAUUSD. Timeframe M5.
     7→
     8→Single visible route: `/` (dashboard with tabbed sections). Backend = Next.js API routes.
     9→Live price feed = mini-service websocket on port 3003.
    10→
    11→## Architecture
    12→- **Frontend**: `/src/app/page.tsx` (single route, client component, tabbed dashboard)
    13→- **Shared types**: `/src/lib/types.ts`

    15→- **DB**: Prisma + SQLite via `@/lib/db`

    17→- **AI**: z-ai-web-dev-sdk LLM for market analysis (server-side only)
    18→- **News**: seed + LLM-summarized; web-search skill optional

    26→## API Contracts (all return JSON)
    27→
    28→### Accounts
    29→- `GET /api/accounts` → `{ accounts: Account[] }`
    30→- `POST /api/accounts` body `{ name, broker, server, login, accountType, currency, leverage, balance }` → `{ account }`
    31→- `PATCH /api/accounts/[id]` body partial → `{ account }`
    32→- `POST /api/accounts/[id]/connect` → `{ account, connected: bool }` (toggle MT5 connection)
    33→- `DELETE /api/accounts/[id]` → `{ ok }`
    34→
    35→### Trades / Positions
    36→- `GET /api/trades?status=open|closed&accountId=&symbol=&limit=` → `{ trades: Trade[] }`
    37→- `POST /api/trades` body `{ accountId, symbol, side, lotSize, stopLoss, takeProfit, source, trailingStop, trailingPips }` → `{ trade }` (opens position at current sim price)
    38→- `POST /api/trades/[id]/close` → `{ trade }` (closes at current sim price, computes pnl)
    39→- `PATCH /api/trades/[id]` body `{ stopLoss?, takeProfit?, trailingStop?, trailingPips? }` → `{ trade }`
    40→
    41→### Orders (pending)
    42→- `GET /api/orders?accountId=` → `{ orders }`
    43→- `POST /api/orders` body `{ accountId, symbol, side, orderType, lotSize, price, stopLoss, takeProfit }` → `{ order }`
    44→- `DELETE /api/orders/[id]` → `{ ok }`
    45→
    46→### Indicators
    47→- `GET /api/indicators` → `{ indicators: Indicator[] }`
    48→- `PATCH /api/indicators/[id]` body `{ enabled?, autoManaged?, weight? }` → `{ indicator }`
    49→- `POST /api/indicators/ai-select` → `{ indicators: Indicator[] }` (AI re-picks enabled set)
    50→
    51→### News
    52→- `GET /api/news?category=&impact=&limit=` → `{ news: NewsItem[] }`
    53→- `POST /api/news/refresh` → `{ news: NewsItem[] }` (uses LLM to synthesize fresh headlines)
    54→
    55→### AI
    56→- `GET /api/ai/signals?symbol=&limit=` → `{ signals: AiSignal[] }`
    57→- `POST /api/ai/analyze` body `{ symbol }` → `{ signal: AiSignal }` (LLM generates fresh signal)
    58→
    59→### Alerts
    60→- `GET /api/alerts` → `{ alerts: Alert[] }`
    61→- `POST /api/alerts` body `{ symbol, condition, price, notifyEmail, message }` → `{ alert }`
    62→- `PATCH /api/alerts/[id]` body `{ active? }` → `{ alert }`
    63→- `DELETE /api/alerts/[id]` → `{ ok }`
    64→
    65→### Logs
    66→- `GET /api/logs?level=&source=&limit=` → `{ logs: Log[] }`
    67→- `POST /api/logs` body `{ level, source, message, stack?, context? }` → `{ log }`
    68→- `DELETE /api/logs` → `{ ok }` (clear all)
    69→
    70→### Backtest
    71→- `GET /api/backtest?symbol=&limit=` → `{ backtests: Backtest[] }`
    72→- `POST /api/backtest` body `{ name, symbol, timeframe, strategy, periodFrom, periodTo, initialCapital }` → `{ backtest }` (simulates)
    73→
    74→### Risk
    75→- `GET /api/risk` → `{ settings: Record<key,value> }`
    76→- `PATCH /api/risk` body `{ settings: Record<key,value> }` → `{ settings }`
    77→- `GET /api/risk/usage` → `{ usedPct, limitPct, openRiskPct, dailyPnlPct, openPositions, maxPositions }`
    78→
    79→### Notifications
    80→- `GET /api/notifications?limit=` → `{ notifications: Notification[] }`
    81→- `POST /api/notifications/test` body `{ recipient? }` → `{ notification }` (creates a test email record, marked sent=true)
    82→
    83→### Dashboard / System
    84→- `GET /api/dashboard` → aggregated `{ accounts, openTrades, todayPnl, riskUsage, sessions, topNews, latestSignals, equitySpark }`
    85→- `GET /api/sessions` → `{ sessions: [{ name, active, progress, nextOpen }] }` (London, NY, Tokyo, Sydney, Overlap)
    86→- `GET /api/symbols` → `{ symbols: [{ symbol, price, changePct, spread, bid, ask, high, low, spark }] }` (current sim snapshot)
    87→- `GET /api/strategies` → `{ strategies: [...] }`
    88→- `GET /api/system/config` → `{ config: Record<key,value> }`
    89→- `PATCH /api/system/config` body `{ config: Record<key,value> }` → `{ config }`

  1087→## Unresolved Issues / Risks
  1088→1. **Auto-trading still enabled**: `autoTradingEnabled=true`. The auto-pilot actively trades every 30s. This is safe (SL/TP auto-close, news-avoidance, risk limits enforced) but users should be aware.
  1089→2. **VLM false negatives**: The GLM-4.6v vision model sometimes reports "No" for features that ARE present (confirmed via DOM checks). DOM-based verification is more reliable than VLM for this dashboard.
  1090→3. **BE trades auto-close quickly**: When BE is applied with a small buffer (2p), the auto-pilot's 5s SL/TP check may close the trade within seconds if price is volatile. This is correct behavior but means BE is best used when price has moved sufficiently in your favor.
  1091→4. **R:R ratio can be < 1:1**: The R:R chart accurately reflects the actual SL/TP distances. If a user sets a wide SL and narrow TP, the R:R will show as unfavorable (e.g., 1:0.5) — this is a feature, not a bug (helps traders spot bad RR).
